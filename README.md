# 📚 engineering-styleguides

Ce dépôt centralise l'ensemble des **normes**, **conventions** et **standards technologiques** de mon écosystème de développement. L'objectif est de garantir une cohérence absolue à travers tous mes projets.

---

## 🎯 1. Vision & Objectifs
Maintenir une rigueur d'ingénierie constante pour transformer des expérimentations en **bases de connaissances industrielles** :
- **Clarté** : Structure identique pour chaque dépôt.
- **Vitesse** : Réduire le temps de setup via des guides actionnables.
- **Qualité** : Application stricte des standards (Clean Code, Contract-First).

## 🏗️ 3. Structure d'un Guide Standard
Pour garantir une lecture rapide, chaque guide présent ici suit ce schéma :
1.  **Philosophie** : Pourquoi ce standard ?
2.  **Mise en œuvre** : Étapes techniques (Commandes, Config).
3.  **Référence Lab** : Lien vers le projet concret (ex : `java-codegen-lab`).
4.  **Standards** : Ce qui est obligatoire vs ce qui est à éviter.

---

## 🚀 Utilisation
Ces guides sont les documents de référence avant le démarrage de tout nouveau projet dans les pôles **Blueprint** ou **Lab**. Ils assurent la cohérence du portfolio avant chaque push sur GitHub.
